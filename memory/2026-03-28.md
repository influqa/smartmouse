## Root Cause Analysis - Complete

### The Core Bug: `subprocess_backend.py` Windows Incompatibility

**Location:** `C:\Users\kevin\.clawteam\ClawTeam-patched\clawteam\spawn\subprocess_backend.py`

**Problem (original code):**
```python
shell_cmd = f"{cmd_str}; {exit_hook}"  # POSIX syntax!
shlex.quote(team_name)  # POSIX single quotes!
subprocess.Popen(shell_cmd, shell=True, ...)  # shell=True = cmd.exe on Windows
stdout=subprocess.DEVNULL  # Errors invisible!
```

**Why it fails on Windows:**
1. `shlex.quote()` uses POSIX single quotes → cmd.exe treats them literally
2. `;` separator → cmd.exe uses `&` for command chaining
3. `stdout=DEVNULL` → Spawn errors invisible
4. Result: Command string invalid, process crashes instantly with no error output

### Key Insight: Windows Shell Compatibility
- Python's `shlex` module is POSIX-only, NOT Windows-compatible
- Windows cmd.exe uses `&` for command chaining, `;` is ignored
- Windows uses double quotes, single quotes are literal characters
- `subprocess.Popen(shell=True)` on Windows uses cmd.exe, NOT bash

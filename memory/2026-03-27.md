## 💡 Reflections & Lessons Learned

### ✅ SOLVED: ClawTeam Windows Agent Persistence Problem
**Root Cause (confirmed via GitHub Issue #81):**
- ClawTeam requires `tmux` for persistent agent sessions (stated in README)
- Windows doesn't have native tmux or proper TTY support
- The `subprocess` backend was "fire-and-forget" - agents exited immediately
- Issue #81: "spawn subprocess agents exit immediately on Windows"
- PR #102 (Mar 26, 2026) addresses some Windows issues but TTY issue still open

**THE FIX I CREATED:**
1. New file: `clawteam/spawn/windows_backend.py` - Persistent worker backend for Windows
   - Creates worker scripts that run in loops (check inbox → process task → sleep → repeat)
   - Uses proper PATH handling for npm-installed `qwen` CLI
   - Logs activity to `.agent_<name>.log` files
   - Parses task list output to pick up tasks properly

2. Modified: `clawteam/spawn/__init__.py` - Added `windows` backend to factory
   ```python
   elif name == "windows":
       from clawteam.spawn.windows_backend import WindowsBackend
       return WindowsBackend()
   ```

3. Modified: `clawteam/cli/commands.py` - Auto-detect Windows and use windows backend
   ```python
   if sys.platform == "win32" and backend is None:
       be_name = "windows"
   ```

4. Created: `WINDOWS_FIX_README.md` - Documentation of the fix
